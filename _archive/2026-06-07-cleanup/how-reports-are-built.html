Methodology

How these reports are built

The standard behind every report in this archive — so the board can trust that the numbers are defined the same way each month and that a page reads correctly no matter when it is opened.

Owner: Sales · Maintained in docs/Reporting-System-Design.md · Updated 7 June 2026

The archive principle

This is a point-in-time archive, not a live dashboard. Every item is dated; the figures live inside each dated report, never on the home page. A published report is closed at publication: corrections happen as restatements in the next report's appendix, never as edits to a published page.

The two-weeks-later test

Every page must read correctly two weeks after publication without being touched. Dated facts only — never "next week", "upcoming" or "we are about to". Next steps are commitments recorded at month close, with a date ("Hold the 2 June call"). Each report carries a snapshot stamp, and its Priorities section states where outcomes are reported: in the next report. That is what lets the archive sit untouched without ever looking dead.

The report spine

Every monthly report follows the same order — the order is the standard. Three tiers serve three audiences in one document:

TierSectionsFor
1 — ExecutiveCover with one-line verdict · executive summary with the asks · scorecard (5 KPIs + stage funnel)Board / CEO — the two-minute read
2 — Operating detailPipeline movement and advancing deals · marketing & events · partnerships · findings · priorities with owner + dateThe commercial team
3 — AppendixFull deal book · early/nurture and backlog tables · CRM snapshot image · sources and restatementsThe analyst

The rules every report passes

  • Two-minute test — verdict, state and ask are readable from the first screen alone.
  • Two-weeks-later test — dated commitments, no relative time, snapshot stamp present.
  • Traceability — every number maps to a metric definition and a dated CRM snapshot.
  • Thin-month honesty — a section with nothing real carries one honest line ("No new partnerships advanced in June."), never filler.
  • Brand voice — dry, operational, literal; the three products never blurred; no unsourced numbers.
  • Restraint — every element either changes a decision or is a tracked monthly metric; otherwise it is cut.

The monthly rhythm

Manual-first by design — no build step, no render engine, no live connector. Producing a month means: snapshot the CRM board → copy the template → fill it from the snapshot, CRM notes, event reports and weekly updates → QA against the rules above → export the PDF to Documents/Reports/ → add the report to the home page. Marketing owns the events/campaigns block and delivers it to a monthly deadline; if it doesn't arrive, the section says so in one line.

Delta now, targets later

Scorecards show value + month-over-month delta only. RAG status and "vs target" switch on once monthly commercial targets are set — not before. Current-looking numbers never appear outside a dated report.
